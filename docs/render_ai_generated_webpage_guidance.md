# Rendering AI‑Generated Webpages from an API

This guide explains how to render a webpage generated by an API in a
client-side application.\
The generated webpage may contain **HTML, CSS, and JavaScript**.

------------------------------------------------------------------------

# Example API Response

Your API might return something like:

``` json
{
  "html": "<div id='app'>Hello</div>",
  "css": "body { font-family: sans-serif; }",
  "js": "document.getElementById('app').onclick = () => alert('Hi');"
}
```

Your client application can then assemble and render these parts.

------------------------------------------------------------------------

# Option 1 --- Render Inside an iframe (Recommended)

Using an iframe isolates the generated code from your main application.

``` html
<iframe id="preview" style="width:100%;height:500px;border:1px solid #ccc;"></iframe>

<script>
async function loadGeneratedPage(prompt) {
  const res = await fetch('/api/generate-page', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt })
  });

  const { html, css, js } = await res.json();

  const fullDoc = `
    <!doctype html>
    <html>
      <head>
        <style>${css}</style>
      </head>
      <body>
        ${html}
        <script>
          ${js}
        <\/script>
      </body>
    </html>
  `;

  document.getElementById('preview').srcdoc = fullDoc;
}
</script>
```

### Advantages

-   CSS is isolated from your app
-   JavaScript cannot interfere with your main application
-   Much safer for AI‑generated content

------------------------------------------------------------------------

# Option 2 --- Inject Directly Into Your Page

You can also inject the generated content into your DOM.

``` html
<div id="generated-root"></div>

<script>
async function loadGeneratedPage(prompt) {
  const res = await fetch('/api/generate-page', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt })
  });

  const { html, css, js } = await res.json();

  const root = document.getElementById('generated-root');

  root.innerHTML = `
    <style>${css}</style>
    ${html}
  `;

  const script = document.createElement('script');
  script.textContent = js;
  root.appendChild(script);
}
</script>
```

### Risks

-   CSS may break your entire site
-   Generated JS can access everything in your page
-   High XSS risk if the content is not trusted

------------------------------------------------------------------------

# Best Practice --- Use a Sandboxed iframe

For AI‑generated or user‑generated prompts, always sandbox the iframe.

``` html
<iframe
  id="preview"
  sandbox="allow-scripts"
  style="width:100%;height:500px;border:1px solid #ccc;"
></iframe>
```

Example loader:

``` javascript
async function loadGeneratedPage(prompt) {
  const res = await fetch('/api/generate-page', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt })
  });

  const { html, css, js } = await res.json();

  const fullDoc = `
    <!doctype html>
    <html>
      <head>
        <meta charset="UTF-8" />
        <style>${css}</style>
      </head>
      <body>
        ${html}
        <script>${js}<\/script>
      </body>
    </html>
  `;

  document.getElementById('preview').srcdoc = fullDoc;
}
```

### Sandbox Notes

`sandbox="allow-scripts"` allows JavaScript execution while preventing
access to the parent page.

You can optionally add permissions such as:

    allow-forms
    allow-popups
    allow-downloads

depending on your needs.

------------------------------------------------------------------------

# If Your API Returns a Full HTML Document

Sometimes the API returns an entire HTML page.

``` json
{
  "document": "<!doctype html><html>...</html>"
}
```

Then rendering becomes simpler:

``` javascript
iframe.srcdoc = response.document;
```

------------------------------------------------------------------------

# Handling External Libraries

If the generated code expects libraries like Chart.js, React, or
Tailwind, you must include them in the iframe document.

Example:

``` html
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
```

------------------------------------------------------------------------

# Security Considerations

Rendering AI‑generated HTML/JS is equivalent to running **untrusted
code**.

You should consider:

-   Using sandboxed iframes
-   Applying Content Security Policy (CSP)
-   Preventing cookie/token leakage
-   Avoiding direct DOM injection
-   Sanitizing HTML if scripts are unnecessary

------------------------------------------------------------------------

# Practical Example

``` html
<input id="prompt" placeholder="Describe a webpage" />
<button id="go">Generate</button>

<iframe
  id="preview"
  sandbox="allow-scripts"
  style="width:100%;height:600px;border:1px solid #ccc;"
></iframe>

<script>
document.getElementById('go').addEventListener('click', async () => {
  const prompt = document.getElementById('prompt').value;

  const res = await fetch('/api/generate-page', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt })
  });

  const { html, css, js } = await res.json();

  const doc = `
    <!doctype html>
    <html>
      <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <style>${css}</style>
      </head>
      <body>
        ${html}
        <script>${js}<\/script>
      </body>
    </html>
  `;

  document.getElementById('preview').srcdoc = doc;
});
</script>
```

------------------------------------------------------------------------

# Summary

Recommended architecture:

1.  API returns `{ html, css, js }` or a full HTML document.
2.  Client assembles the document.
3.  Render it inside a **sandboxed iframe**.
4.  Avoid direct DOM injection unless content is fully trusted.

This approach keeps your application safe while still allowing dynamic
webpage generation.
